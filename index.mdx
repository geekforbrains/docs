# Local agents. Normal files. Real work.

Operator is a multi-agent runtime that lives in `~/.operator/`. Connect agents to Slack, give them tools, schedule jobs, teach them skills — all with plain files you can read, edit, and version control.

No dashboards. No black boxes. Just markdown and YAML.

## Everything is a file

```text
~/.operator/
├── operator.yaml           # config
├── .env                    # secrets
├── SYSTEM.md               # global prompt
├── agents/
│   └── <agent>/
│       ├── AGENT.md        # agent charter
│       ├── workspace/      # working files
│       ├── memory/         # what it knows
│       └── state/          # machine bookkeeping
├── jobs/
│   └── <job>/JOB.md        # scheduled work
├── skills/
│   └── <skill>/SKILL.md    # reusable know-how
├── memory/
│   ├── global/             # shared knowledge
│   └── users/              # per-user knowledge
├── shared/                 # cross-agent exchange
├── db/                     # SQLite (history, index)
└── logs/
```

Prompts live in git. Memory is readable markdown. Jobs and skills are plain files, not hidden state. If something goes wrong, you can open a file and see exactly why.

## Pick the right primitive

| Primitive | Reach for it when you want… |
| --- | --- |
| `AGENT.md` | a distinct role, mission, or personality |
| `permissions` | to widen or narrow what an agent can do |
| `rules/` | standing instructions injected into every run |
| `notes/` | durable knowledge searched on demand |
| `state/` | machine bookkeeping — cursors, queues, checkpoints |
| `skills/` | reusable instructions, scripts, and references |
| `jobs/` | recurring work on a cron schedule |
