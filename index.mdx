# Agents have joined the chat.

Operator is a local agent runtime for teams. It connects agents to Slack, gives them real tools, and keeps the important parts of the system in normal files under `~/.operator/`.

That design is the product surface, not an implementation detail. Prompts live in markdown. Jobs live in markdown. Skills live in markdown. Memory is readable on disk. SQLite stores conversation history, user identities, and search indexes, but the human-facing source of truth stays in files you can inspect and edit.

## What Lives On Disk

```text
~/.operator/
├── operator.yaml
├── .env
├── SYSTEM.md
├── agents/
│   └── <agent>/
│       ├── AGENT.md
│       ├── workspace/
│       ├── memory/
│       └── state/
├── jobs/
│   └── <job>/
│       └── JOB.md
├── skills/
│   └── <skill>/
│       └── SKILL.md
├── memory/
│   ├── global/
│   └── users/
├── shared/
├── db/
└── logs/
```

Why it matters:

- `AGENT.md` is the agent charter.
- `JOB.md` is the recurring task definition.
- `SKILL.md` is reusable know-how.
- memory files are durable knowledge, split into `rules/` and `notes/`.
- workspace files are work artifacts, not hidden model state.

## What Operator Does

- Runs one or more named agents from `operator.yaml`.
- Authenticates humans through stored transport identities such as `slack:U12345`.
- Builds a system prompt from `SYSTEM.md`, the active `AGENT.md`, discovered skills, peer-agent descriptions, transport context, and active rules.
- Lets the agent call built-in tools plus transport-specific Slack tools.
- Stores chat history and runtime bookkeeping in SQLite.
- Runs scheduled jobs once per minute from `jobs/<name>/JOB.md`.

## Why The File-First Model Works

- It is reviewable. You can diff agent prompts, job changes, and skill updates in git.
- It is debuggable. If a rule exists, you can read the exact file that produced it.
- It is recoverable. You do not need a hidden dashboard to inspect state after something goes wrong.
- It keeps memory honest. Durable knowledge is visible, editable, and explicitly scoped.
