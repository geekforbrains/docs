The memories CLI is read-only inspection.

## Commands

```sh
operator memories [--scope/-s SCOPE] [--scope-id/-i ID] [--pinned] [--limit/-n N]
operator memories stats
```

These commands read the SQLite store directly and work even when runtime memory services are currently disabled.

## `operator memories`

```sh
operator memories
operator memories --scope agent --scope-id operator
operator memories --scope global --scope-id global --pinned
```

Options:

| Option | Meaning |
| --- | --- |
| `--scope`, `-s` | `user`, `agent`, or `global` |
| `--scope-id`, `-i` | specific scope identifier |
| `--pinned` | only show pinned memories |
| `--limit`, `-n` | max rows, default `50` |

Displayed columns:

- `ID`
- `Scope`
- `Retention`
- `Expires`
- `Content`
- pin marker

Expired candidate memories do not appear in the active listing.

## `operator memories stats`

```sh
operator memories stats
```

Shows counts per scope:

- total
- candidate
- durable
- pinned

This is the quickest way to see whether memory growth is concentrated in one agent or scope.

## Debugging Workflow

1. Confirm the memory exists:

   ```sh
   operator memories --scope agent --scope-id operator
   ```

2. Check scope-level totals:

   ```sh
   operator memories stats
   ```

3. Inspect live recall logs:

   ```sh
   operator logs -f | rg 'memory recall:|memory injection:'
   ```
