---
title: "Removed: `operator kv`"
description: "Compatibility note for an older CLI page"
---

`operator kv` is not part of the current CLI.

For operational state, current Operator uses per-agent JSON state files and runtime state tools instead:

- `get_state`
- `set_state`
- `append_state`
- `pop_state`
- `list_state`
- `delete_state`

Those files live under `~/.operator/agents/<agent>/state/`.

See [Built-in Tools](/concepts/tools) for the current state-tool surface.
