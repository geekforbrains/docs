The CLI skill commands are intentionally narrow.

## Commands

```sh
operator skills
operator skills list
```

The CLI does **not** create, edit, or reset skills. Use the filesystem or the runtime tools `create_skill`, `update_skill`, `delete_skill`, and `list_skills` for that.

## `skills` / `skills list`

```sh
operator skills
operator skills list
```

Both commands do the same thing.

They show:

- skill name
- description
- env status:
  - `ok`
  - `missing: ...`
  - `-`

This is the fastest way to confirm that a skill is discoverable and whether its required environment variables are present.

## Typical Workflow

```sh
operator skills
operator logs -n 50
```

If a skill is missing from the list:

- confirm `~/.operator/skills/<name>/SKILL.md` exists
- confirm the frontmatter `name` matches the directory
- check logs for validation warnings
