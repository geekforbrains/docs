The CLI skill commands are intentionally narrow.

## Commands

```sh
operator skills
operator skills list
operator skills reset <name>
operator skills reset --all
```

The CLI does **not** create or edit arbitrary skills. Use the filesystem or the `manage_skill` tool for that.

## `skills` / `skills list`

```sh
operator skills
operator skills list
```

Shows:

- skill name
- description
- env status:
  - `ok`
  - `missing: ...`
  - `-`

This is the fastest way to confirm that a skill is discoverable and whether its required environment variables are present.

## `skills reset`

```sh
operator skills reset skill-creator
operator skills reset --all
```

This only applies to bundled skills shipped with Operator.

Behavior:

- deletes the local bundled skill directory
- copies the original packaged version back in place

If you run `operator skills reset` with no arguments, the CLI prints the available bundled skill names and usage help.

<Warning>
  Resetting a bundled skill discards local edits in that skill directory.
</Warning>

## Bundled Skills

Current bundled skills:

| Skill | Purpose |
| --- | --- |
| `job-creator` | helps agents create and update jobs correctly |
| `skill-creator` | helps agents create and refine skills correctly |

Missing bundled skills are also auto-installed during `operator init` and runtime startup.
