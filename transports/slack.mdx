Operator uses Slack Socket Mode. You do not need a public webhook URL.

## Scan First

- Supported inbound patterns: DMs and `@mentions` in invited channels
- Supported files: inline images up to 5 MB, all other files saved locally up to 50 MB
- The runtime uses your configured timezone when formatting Slack timestamps
- Slack adds transport tools like `list_channels`, `read_channel`, and `read_thread`
- In-chat commands start with `!` and bypass the LLM

## Slack App Setup

### 1. Create the app

Create a Slack app and enable **Socket Mode**.

Create an app-level token with:

- `connections:write`

That becomes your `SLACK_APP_TOKEN`.

### 2. Add bot scopes

Under **OAuth & Permissions**, add:

- `app_mentions:read`
- `channels:history`
- `channels:read`
- `chat:write`
- `files:read`
- `files:write`
- `groups:history`
- `groups:read`
- `im:history`
- `im:read`
- `im:write`
- `users:read`

### 3. Subscribe to events

Subscribe to:

- `app_mention`
- `message.im`

That covers:

- DMs
- channel mentions

### 4. Install the app

Install the app to your workspace and copy the bot token:

- `SLACK_BOT_TOKEN`

### 5. Configure Operator

Add the tokens to `~/.operator/.env`:

```sh
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
```

Then configure the agent transport:

```yaml
agents:
  operator:
    transport:
      type: slack
      bot_token_env: SLACK_BOT_TOKEN
      app_token_env: SLACK_APP_TOKEN
```

Optional overrides:

```yaml
agents:
  operator:
    transport:
      type: slack
      bot_token_env: SLACK_BOT_TOKEN
      app_token_env: SLACK_APP_TOKEN
      include_archived_channels: false
      inject_channels_into_prompt: false
      channel_cache_ttl_seconds: 900
      warm_channels_on_startup: true
```

## What The Runtime Handles

SlackTransport accepts:

- DMs from human users
- `@mentions` in channels where the bot is invited

It ignores:

- bot-authored messages
- most Slack system/edit subtypes

## Channel and Thread Context

Slack runtime behavior includes:

- resolving user IDs to human names
- resolving channel IDs to `#channel-name`
- formatting Slack timestamps in your configured timezone
- exposing channel-reading tools to the agent
- warming an in-memory per-bot channel cache on startup by default
- refreshing that cache lazily on demand after TTL expiry or lookup miss
- excluding archived channels from discovery unless you opt in

When the agent is first mentioned inside an existing thread, Operator may inject a one-time snapshot of earlier replies in that thread so the model has context it missed before being added.

## In-Chat Commands

Commands bypass the LLM and are handled directly by the runtime.

| Command | Behavior |
| --- | --- |
| `!help` | list available commands |
| `!stop` | cancel the active request in the current conversation |
| `!restart` | returns a message telling you to use the CLI |
| `!agents` | list configured agents |
| `!jobs` | list jobs |
| `!jobs <name>` | show one job's details |
| `!skills` | list discovered skills |
| `!memories` | list pinned agent/global memories |

There is currently **no** `!config` command.

## Conversation Behavior

Slack conversation IDs are built from:

```text
slack:<agent-name>:<channel-id>:<root-thread-ts>
```

If a second message arrives while the same conversation is still busy, Operator replies:

```text
Still processing a request. Send `!stop` to cancel it.
```

## Slack Transport Tools

These tools exist only during live Slack-backed runs:

| Tool | Use it for |
| --- | --- |
| `list_channels(query="")` | list postable channels, IDs, and cached topic/purpose snippets, optionally filtered by substring |
| `read_channel(channel, count=20)` | fetch recent channel messages |
| `read_thread(channel, thread_id, count=50)` | fetch thread replies |

`send_message` also accepts Slack channel names like `#general` or raw channel IDs.

By default, Slack channel names are not injected into the system prompt. Use `list_channels()` when the agent needs to inspect destinations first, or set `inject_channels_into_prompt: true` for a specific bot if you want cached channel names included in prompt extras.

## File Handling

Inbound Slack files behave like this:

| File type | Runtime behavior |
| --- | --- |
| image/png, image/jpeg, image/gif, image/webp up to 5 MB | inlined into the model request as an image block |
| other file types up to 50 MB | saved under `workspace/uploads/` and described in text |
| larger than 50 MB | skipped |

Additional details:

- duplicate filenames get suffixed (`file_1.txt`, etc.)
- unsafe filenames are sanitized to a basename
- saved files can be read later with workspace tools

Outbound uploads use `send_file`, which calls Slack `files.upload_v2`.
