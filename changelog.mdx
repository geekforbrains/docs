## Unreleased

## 0.6.0 · 2026-03-06

### Added

- Guided onboarding via `operator setup`
- `python -m operator_ai` entrypoint
- Simple reasoning control via `thinking: off|low|medium|high`
- Memory retention classes: `candidate` and `durable`

### Changed

- Runtime/process settings now live under `runtime` in `operator.yaml`
- Current time is injected just in time for live model requests instead of being stored in history
- Jobs and sub-agents inherit resolved agent thinking, runtime prompt, and skill filter more consistently
- Memory recall is stricter about scope and retention
- Legacy memory tables reset on first run of this release to support retention and expiry metadata
- Harvester and cleaner prompts are stricter about short-lived conversational noise

### Fixed

- `run_skill` expands env var references after env sanitization while preserving `$OPERATOR_HOME`
- Cross-provider fallbacks drop reasoning metadata before each model call
- Config loading raises `ConfigError` internally instead of terminating via `SystemExit`
- Slack thread timestamps render in the configured runtime timezone
- Same-agent `spawn_agent(...)` once again inherits the current agent's full prompt/runtime context

## 0.5.1 · 2026-03-05

### Fixed

- `operator service install` now embeds the current interactive `PATH` so background services can find pyenv/nvm/Homebrew tools
- Cleaner error output for simple `FileNotFoundError` cases

### Changed

- `.env` uses `setdefault` semantics: exported shell env vars win
- `operator init` writes API-key placeholders only; PATH is handled by the service definition

## 0.5.0 · 2026-03-05

### Added

- Agent frontmatter with `name` and `description`
- Cross-agent spawning via `spawn_agent(agent="...")`
- Spawn logging with parent, target, and depth
- `$OPERATOR_HOME` in shell commands, skill scripts, and job hooks
- `job-creator` bundled skill
- Prompt templates extracted to `.md` files under `prompts/`

### Changed

- `AGENT.md` frontmatter is stripped before prompt injection
- `operator agents` shows frontmatter descriptions
- Cross-agent sub-runs respect the target agent's sandbox and permissions
- System prompt trimmed and reworded for clarity

### Fixed

- `!stop` cancels the active task immediately
- Incomplete tool-call history is trimmed on cancel/load
- Cancel state is cleared after a stopped run

## 0.4.0 · 2026-03-05

### Added

- Users, roles, and transport-message auth
- Agent permissions for tools and skills
- Skill access tools: `read_skill` and `run_skill`
- Agent sandboxing for file tools
- Slack file/media support
- `send_file`
- `operator job run`
- `manage_users` admin-only tool

### Fixed

- `send_message` and `send_file` default to the current conversation channel/thread
- Hook scripts respect their shebang interpreter
- Skill script paths expand correctly in `run_skill`
- `manage_skill list` respects agent skill permissions
- Login shell PATH handling replaced with more reliable env-file/service behavior
