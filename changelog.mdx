## 0.5.0

### Added

- **Agent frontmatter** тАФ `AGENT.md` files now support optional YAML frontmatter with `name` and `description`. Agents with descriptions are automatically listed in each other's system prompts for inter-agent awareness.
- **Cross-agent spawning** тАФ `spawn_agent` accepts an optional `agent` parameter to delegate tasks to a different agent with its own prompt, models, workspace, sandbox, and permissions.
- **Spawn logging** тАФ every `spawn_agent` call is logged with the parent agent, target agent, and nesting depth for traceability.

### Changed

- `AGENT.md` frontmatter is stripped before prompt injection тАФ the LLM only sees the markdown body.
- `operator agents` CLI now shows a Description column from agent frontmatter.
- Sub-agent sandbox and permissions inheritance now respects the target agent's config when spawning a different agent.

## 0.4.1

### Added

- **`OPERATOR_HOME` env var** тАФ the resolved Operator base directory is now exposed as `$OPERATOR_HOME` in all shell commands, skill scripts, and job hooks. Use it instead of `~/.operator` for reliable path resolution in non-shell contexts.
- **`job-creator` bundled skill** тАФ teaches agents how to create and manage scheduled jobs using `manage_job`, covering JOB.md anatomy, cron schedules, prompt patterns, KV state, hooks, and the `send_message` delivery rule.
- **Prompts extracted to files** тАФ system, agent, job, and subagent prompts moved from inline strings to `.md` files under `prompts/`, loaded via `load_prompt()`.

### Changed

- System prompt trimmed and reworded for clarity
- `$OPERATOR_HOME` path documented in agent context and job context prompts

## 0.4.0

### Added

- **Users, roles, and auth** тАФ every inbound message is authenticated against a user database with role-based agent access control
- **Agent permissions** тАФ flat allow-lists for [tools and skills](/concepts/agents#permissions) per agent
- **Skill access tools** тАФ [`read_skill` and `run_skill`](/concepts/tools#skill-access) let restricted agents use skills without shell access
- **Agent sandbox** тАФ [`sandbox`](/concepts/agents#sandbox) config flag confines file tools to the workspace (default) or grants full filesystem access
- **File support** тАФ users can upload images, PDFs, and documents in Slack; images are sent to the model as vision input, other files are saved to `workspace/uploads/`
- **`send_file` tool** тАФ agents can upload files back to chat
- **`operator job run`** тАФ run jobs from the CLI with dedicated transport and logging
- **`manage_users` tool** тАФ admin-only runtime user management

### Fixed

- `send_message` and `send_file` default to the current conversation's channel and thread
- Hook scripts respect shebang interpreter
- Skill script paths expanded correctly for `run_skill`
- `manage_skill list` respects agent skill permissions

## 0.3.0

### Added

- **Agent permissions** тАФ flat allow-lists for tools and skills per agent
- **Shared directory** тАФ `~/.operator/shared/` is symlinked into every agent workspace for cross-agent file sharing
- **Bundled skills** тАФ skills shipped with the package, auto-installed on `operator init`
- **`manage_skill` tool** тАФ agents can create, update, and delete skills at runtime
- **`operator skills reset`** тАФ CLI command to restore bundled skills to their original version
- **Transport-scoped read tools** тАФ Slack transport adds `list_channels`, `read_channel`, and `read_thread`
- **Timezone config** тАФ `defaults.timezone` sets the IANA timezone for timestamps and job scheduling (defaults to UTC)

### Fixed

- Default agent renamed from `hermy`/`default` to `operator`

## 0.2.2

### Fixed

- **Prompt caching** тАФ system prompt is split into a stable prefix and dynamic suffix with Anthropic cache breakpoints so the prefix is reused across turns
- Rolling cache breakpoint on conversation history (penultimate user message) for multi-turn token savings
- Read OpenAI `cached_tokens` from `prompt_tokens_details` for unified cache reporting

### Added

- Per-run ID logging via ContextVar for tracing agent runs in logs
- Usage line now shows cache write tokens, prefixed with `Usage:`

## 0.2.1

### Added

- Per-job model override via `model` field in [JOB.md frontmatter](/concepts/jobs#model-override)

## 0.2.0

### Added

- [`operator init`](/cli/overview#setup) command to scaffold `~/.operator/` with starter config and agent

## 0.1.0

### Added

- Initial public beta release
- LiteLLM-backed agent with model fallback chains
- Slack transport with Socket Mode and thread continuity
- SQLite persistence with WAL mode
- Scheduled jobs with cron, prerun gating, and postrun hooks
- Vector memory with harvesting and semantic search
- Key-value store with TTL support
- Built-in tools (shell, file, web, memory, KV, jobs, messaging, sub-agents) plus transport-specific tools
- Skills discovery from markdown files
- Turn-safe context truncation
- CLI for inspection, service management, and job control
