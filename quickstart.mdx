## Install

```sh
pip install operator-ai
```

Or install from source:

```sh
git clone https://github.com/geekforbrains/operator.git
cd operator
pip install -e .
```

## Guided Setup

```sh
operator setup --run
```

This is the fastest path from a clean install to a running Slack-backed agent. It:
- scaffolds `~/.operator/`
- asks which model provider you want to use
- detects your local timezone and lets you confirm or change it
- saves provider and Slack credentials into `runtime.env_file`
- creates your first admin user
- starts the runtime immediately with `--run`

If the `operator` script is not on your `PATH` right after install, use:

```sh
python3 -m operator_ai setup --run
```

## What Setup Creates

Setup leaves you with the same file layout as `operator init`:
- `operator.yaml` ŌĆö runtime config
- `.env` ŌĆö API key file (add your keys here)
- `SYSTEM.md` ŌĆö system preamble
- `agents/operator/AGENT.md` ŌĆö your first agent
- `skills/` ŌĆö bundled starter skills

The scaffold step is idempotent ŌĆö existing files are not overwritten.

## Manual Setup

If you want to configure everything yourself instead, run:

```sh
operator init
```

Then edit `~/.operator/operator.yaml` with your runtime, model, and transport:

```yaml
runtime:
  timezone: "America/Vancouver"
  env_file: ".env"
  show_usage: false
  reject_response: ignore

defaults:
  models:
    - "anthropic/claude-sonnet-4-6"
  max_iterations: 25

agents:
  operator:
    transport:
      type: slack
      bot_token_env: SLACK_BOT_TOKEN
      app_token_env: SLACK_APP_TOKEN
```

Set your API keys in `~/.operator/.env` (created by `operator setup` or `operator init`):

```sh
ANTHROPIC_API_KEY=sk-ant-...
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
```

## Add yourself (manual path)

Skip this if you used `operator setup` ŌĆö it already created your first admin user.

If you used the manual path, register yourself as an admin so Operator knows who you are:

```sh
operator user add yourname --role admin slack YOUR_SLACK_USER_ID
```

## Run it

```sh
operator
```

If you used `operator setup --run`, the runtime is already started. Otherwise, run `operator` now.

That's it. Message your agent in Slack. ­¤ÉÆ

Logs go to `~/.operator/logs/operator.log`.

## Run as a service

Want it running in the background?

```sh
operator service install   # install the service definition
operator service start     # start it when needed
```

On macOS, `operator service install` writes and loads `~/Library/LaunchAgents/ai.operator.plist`. On Linux, it writes `~/.config/systemd/user/operator.service` and enables it.

See the [Service CLI](/cli/service) for more details.
