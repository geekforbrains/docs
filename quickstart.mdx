This is the shortest path to a working install. Start with the generated layout, prove the runtime works, then tighten permissions and add more agents.

## Fast Path

1. Install the package and scaffold `~/.operator/`.
2. Add your model API key and Slack tokens.
3. Create your first admin user.
4. Start the runtime.
5. DM the bot or mention it in Slack.

## 1. Install And Scaffold

```sh
pip install operator-ai
operator init
```

If the `operator` script is not on your `PATH` yet:

```sh
python3 -m operator_ai init
```

`operator init` writes the starter files:

- `~/.operator/operator.yaml`
- `~/.operator/.env`
- `~/.operator/SYSTEM.md`
- `~/.operator/agents/operator/AGENT.md`

It also creates the default layout for jobs, skills, shared files, logs, memory, state, and the starter `operator` agent workspace.

The generated config is intentionally permissive for the first run:

- one `operator` agent
- full tool access
- full skill access
- Slack transport wired through env vars
- starter `permission_groups` you can reuse later

If `operator.yaml` already exists, `operator init` prompts before overwriting it. Use `operator init --force` to replace `operator.yaml` without touching your existing `.env` or `AGENT.md` files.

## 2. Add Secrets

Put your secrets in `~/.operator/.env`:

```sh
ANTHROPIC_API_KEY=sk-ant-...
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
```

The generated starter config points the default agent at Slack and uses the first model in `defaults.models`:

```yaml
defaults:
  models:
    - "anthropic/claude-sonnet-4-6"
  thinking: "off"

agents:
  operator:
    permissions:
      tools: "*"
      skills: "*"
    transport:
      type: slack
      env:
        bot_token: SLACK_BOT_TOKEN
        app_token: SLACK_APP_TOKEN
      settings:
        include_archived_channels: false
        inject_channels_into_prompt: true
        inject_users_into_prompt: true
        expand_mentions: true
```

`runtime.env_file` is loaded with `setdefault` semantics, so variables already present in your shell win over values from `.env`.

## 3. Create The Slack App

Operator currently runs on Slack only.

The short version:

- enable Socket Mode
- create an app-level token with `connections:write`
- create a bot token
- subscribe to `app_mention`, `message.im`, `reaction_added`, and `reaction_removed`

Use the full [Slack guide](/transports/slack) for the exact scopes and behavior.

## 4. Create The First Admin User

Operator does not auto-create users. Until you add one, inbound messages are rejected.

Create yourself as the first admin:

```sh
operator user add yourname --role admin slack YOUR_SLACK_USER_ID
```

That command creates the user, assigns the `admin` role, and links the Slack identity in one step.

## 5. Start The Runtime

Foreground:

```sh
operator
```

Background service:

```sh
operator service install
operator service start
```

Platform detail:

- macOS: `service install` writes and loads a `launchd` plist immediately
- Linux: `service install` writes the user unit and enables it, then `service start` starts it

## 6. Send The First Message

Once the runtime is running:

1. invite the Slack app to a channel if you want channel access
2. DM the bot or mention it in a channel
3. send something simple like `hello`

Slack sessions are intentionally thread-scoped:

- a top-level DM starts a new session
- a top-level channel mention starts a new thread session
- later replies count as new input only when they mention the agent again

If you send `stop` or `cancel`, Operator cancels the in-flight request for that conversation.

## 7. Run The First Checks

```sh
operator config
operator agents
operator tools
operator logs -n 50
```

These verify that:

- config parses
- the agent exists
- built-in tools are registered
- logs look normal
