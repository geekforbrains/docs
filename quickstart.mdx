This is the shortest path to a working Operator install. Keep the starter setup simple, prove the runtime works, then tighten permissions and add more agents.

## Fast Path

1. Install the package and scaffold `~/.operator/`.
2. Add a model API key and Slack tokens to `.env`.
3. Create your first admin user.
4. Start the runtime.
5. DM the bot or mention it in Slack.

## 1. Install And Scaffold

```sh
pip install operator-ai
operator init
```

If the `operator` script is not on your `PATH` yet:

```sh
python3 -m operator_ai init
```

`operator init` creates the default home under `~/.operator/` and writes:

- `operator.yaml`
- `.env`
- `SYSTEM.md`
- `agents/operator/AGENT.md`
- the default jobs, skills, shared, db, logs, workspace, memory, and state directories

It also writes starter `permission_groups` and a single Slack-connected `operator` agent with full tool and skill access so you can get a first run working quickly.

If `operator.yaml` already exists, the command prompts before overwriting it. `operator init --force` overwrites `operator.yaml` without prompting, but keeps existing `.env` and `AGENT.md` files.

## 2. Create The Slack App

Operator currently supports Slack as its runtime transport.

Create a Slack app, enable Socket Mode, add the required bot scopes, subscribe to the required events, and install the app to your workspace. The full setup is on the [Slack guide](/transports/slack). The short version is:

- create an app-level token with `connections:write`
- create a bot token
- subscribe to `app_mention`, `message.im`, `reaction_added`, and `reaction_removed`

## 3. Add Secrets

Put the secrets in `~/.operator/.env`:

```sh
ANTHROPIC_API_KEY=sk-ant-...
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
```

Use the provider key that matches your configured model IDs. The generated config defaults to:

```yaml
defaults:
  models:
    - "anthropic/claude-sonnet-4-6"
```

The default config also wires the starter agent to Slack:

```yaml
agents:
  operator:
    permissions:
      tools: "*"
      skills: "*"
    transport:
      type: slack
      env:
        bot_token: SLACK_BOT_TOKEN
        app_token: SLACK_APP_TOKEN
      settings:
        include_archived_channels: false
        inject_channels_into_prompt: true
        inject_users_into_prompt: true
        expand_mentions: true
```

<Note>
  `runtime.env_file` is loaded with `setdefault` semantics. Values already present in your shell environment win over values from `.env`.
</Note>

## 4. Create The First Admin User

Operator does not auto-create users. Until you add one, inbound Slack messages are rejected.

Create yourself as the first admin:

```sh
operator user add yourname --role admin slack YOUR_SLACK_USER_ID
```

That command creates the user record, assigns the `admin` role, and links the Slack identity in one step.

## 5. Start The Runtime

Foreground:

```sh
operator
```

Background service:

```sh
operator service install
operator service start
```

Platform detail:

- macOS: `service install` writes the plist and loads it immediately.
- Linux: `service install` writes the user unit and enables it, but you still need `operator service start`.

## 6. Send The First Message

Once the runtime is running:

1. invite the Slack app to a channel if you want channel access
2. DM the bot or `@mention` it in an invited channel
3. send a simple message such as `hello`

Slack sessions are intentionally thread-scoped:

- a top-level DM starts a new session
- a top-level channel mention starts a new session thread
- later channel replies count as new input only when they mention the agent again

If the agent needs context outside that thread, it uses Slack tools such as `slack_read_channel` or `slack_read_thread`.

## 7. Run The Basic Checks

```sh
operator config
operator agents
operator tools
operator logs -n 50
```

These confirm that:

- config parses
- the configured agents are visible
- built-in tools are registered
- the runtime is logging normally
