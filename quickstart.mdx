## Fast Path

```sh
pip install operator-ai
operator setup --run
```

That path:

- scaffolds `~/.operator/`
- detects your local timezone and writes it to `operator.yaml`
- asks for an LLM provider and API key
- asks for Slack tokens and your Slack user ID
- creates your first admin user
- starts the runtime immediately

If the `operator` script is not on your `PATH` yet, use:

```sh
python3 -m operator_ai setup --run
```

## Requirements

- Python `3.11+`
- A Slack app using Socket Mode
- One model provider API key:
  - `ANTHROPIC_API_KEY`
  - `OPENAI_API_KEY`
  - `GEMINI_API_KEY` or `GOOGLE_API_KEY`

## What `operator setup` Asks For

| Prompt | Notes |
| --- | --- |
| Model provider | `anthropic`, `openai`, or `gemini` |
| Timezone | Must be a valid IANA zone like `America/Vancouver` |
| Provider API key | Saved into `~/.operator/.env` unless already present |
| `SLACK_BOT_TOKEN` | Bot token, usually `xoxb-...` |
| `SLACK_APP_TOKEN` | App-level Socket Mode token, usually `xapp-...` |
| Slack user ID | Your admin identity, like `U123ABC45` |
| Admin username | Lowercase letters, numbers, dots, and hyphens only |

`operator setup` also accepts flags so you can run it non-interactively:

```sh
operator setup \
  --provider openai \
  --timezone America/Vancouver \
  --username gavin \
  --api-key sk-... \
  --slack-bot-token xoxb-... \
  --slack-app-token xapp-... \
  --slack-user U123ABC45 \
  --run
```

Use `--force` if secrets already exist and you want to re-prompt or overwrite them.

## What Setup Writes

After `operator setup` or `operator init`, you will have:

- `~/.operator/operator.yaml`
- `~/.operator/.env`
- `~/.operator/SYSTEM.md`
- `~/.operator/agents/operator/AGENT.md`
- `~/.operator/agents/operator/workspace/`
- `~/.operator/shared/`
- `~/.operator/jobs/`
- `~/.operator/skills/` with bundled starter skills

The scaffold step is idempotent: existing files are not overwritten.

<Note>
  The `.env` loader uses `setdefault` semantics. Existing shell environment variables win over values from `runtime.env_file`.
</Note>

## Manual Setup

If you want the scaffold only:

```sh
operator init
```

The starter config generated by `operator init` includes a working Slack transport block, `defaults.context_ratio: 0.5`, a starter `guest` role, and `defaults.max_iterations: 50`.

A minimal manual config looks like this:

```yaml
runtime:
  timezone: "America/Vancouver"
  env_file: ".env"
  show_usage: false
  reject_response: ignore

defaults:
  models:
    - "anthropic/claude-sonnet-4-6"
  thinking: off
  max_iterations: 25
  context_ratio: 0.5

agents:
  operator:
    transport:
      type: slack
      bot_token_env: SLACK_BOT_TOKEN
      app_token_env: SLACK_APP_TOKEN

roles:
  guest:
    agents: []
```

Add secrets to `~/.operator/.env`:

```sh
ANTHROPIC_API_KEY=sk-ant-...
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
```

Register yourself as the first admin:

```sh
operator user add yourname --role admin slack YOUR_SLACK_USER_ID
```

Then start the runtime:

```sh
operator
```

## First Slack Message

Once the runtime is running:

1. Invite the Slack app to a channel if you want channel access.
2. DM the bot, or `@mention` it in an invited channel.
3. Send a simple first message like `hello`.

If the user is unknown, Operator rejects the message. `operator setup` avoids that by creating your first admin identity up front.

## Run In The Background

```sh
operator service install
operator service start
```

- On macOS, this writes and loads `~/Library/LaunchAgents/ai.operator.plist`.
- On Linux, this writes `~/.config/systemd/user/operator.service` and enables it. Start it explicitly with `operator service start`.

`service install` captures your current shell `PATH`, which matters if your agents need tools from Homebrew, pyenv, nvm, or similar setups.

## Sanity Checks

```sh
operator config
operator agents
operator tools
operator logs -n 50
```

These confirm that config loads, agents are visible, tools are registered, and the runtime has started logging.
