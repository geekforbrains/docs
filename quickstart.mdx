## Fast Path

```sh
pip install operator-ai
operator init
```

That path:

- scaffolds the full `~/.operator/` layout
- writes a Slack-focused starter `operator.yaml`
- writes `.env`, `SYSTEM.md`, and `agents/operator/AGENT.md`
- preserves existing `.env` and `AGENT.md` files
- prompts before overwriting an existing `operator.yaml`

If the `operator` script is not on your `PATH` yet, use:

```sh
python3 -m operator_ai init
```

## Requirements

- Python `3.11+`
- A Slack app using Socket Mode
- One model provider API key:
  - `ANTHROPIC_API_KEY`
  - `OPENAI_API_KEY`
  - `GEMINI_API_KEY` or `GOOGLE_API_KEY`
- Slack bot and app tokens
- Your Slack user ID for the first admin user

## What `operator init` Writes

After `operator init`, you will have:

- `~/.operator/operator.yaml`
- `~/.operator/.env`
- `~/.operator/SYSTEM.md`
- `~/.operator/agents/operator/AGENT.md`
- `~/.operator/agents/operator/workspace/{inbox,work,artifacts,tmp}`
- `~/.operator/agents/operator/memory/{rules,notes,trash}`
- `~/.operator/agents/operator/state/`
- `~/.operator/shared/`
- `~/.operator/jobs/`
- `~/.operator/skills/`
- `~/.operator/memory/global/`
- `~/.operator/memory/users/`
- `~/.operator/db/`
- `~/.operator/logs/`

If `operator.yaml` already exists, `operator init` asks before replacing it. Use `--force` to overwrite it without a prompt.

<Note>
  The `.env` loader uses `setdefault` semantics. Existing shell environment variables win over values from `runtime.env_file`.
</Note>

## Edit `operator.yaml`

The starter config is Slack-only and gives the default `operator` agent full tool and skill access:

```yaml
runtime:
  env_file: ".env"
  show_usage: false
  reject_response: ignore

defaults:
  models:
    - "anthropic/claude-sonnet-4-6"
  thinking: "off"
  max_iterations: 25
  context_ratio: 0.5
  hook_timeout: 30

agents:
  operator:
    permissions:
      tools: "*"
      skills: "*"
    transport:
      type: slack
      env:
        bot_token: SLACK_BOT_TOKEN
        app_token: SLACK_APP_TOKEN
      settings:
        include_archived_channels: false
        inject_channels_into_prompt: true
        inject_users_into_prompt: true
        expand_mentions: true

roles:
  guest:
    agents: []
```

`operator init` also writes starter `permission_groups` for common tool clusters like files, web, jobs, skills, memory, users, and sub-agents.

## Add Secrets

Add the required keys to `~/.operator/.env`:

```sh
ANTHROPIC_API_KEY=sk-ant-...
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
```

## Create The First Admin

Register yourself as the first admin:

```sh
operator user add yourname --role admin slack YOUR_SLACK_USER_ID
```

`operator init` does not create users for you. Until you add at least one user, inbound Slack messages will be rejected.

## Start The Runtime

Then start the runtime:

```sh
operator
```

Or run it in the background:

```sh
operator service install
operator service start
```

On macOS, `service install` loads the service immediately. On Linux, it only enables the unit, so `operator service start` is still required.

## First Slack Message

Once the runtime is running:

1. Invite the Slack app to a channel if you want channel access.
2. DM the bot, or `@mention` it in an invited channel.
3. Send a simple first message like `hello`.

If your Slack user is not registered, Operator rejects the message according to `runtime.reject_response`.

Slack sessions are thread-scoped on purpose. A top-level DM or top-level channel `@mention` starts a fresh session thread. In channels, mention the agent again in later thread replies if you want those replies treated as new input.

## Sanity Checks

```sh
operator config
operator agents
operator tools
operator logs -n 50
```

These confirm that config loads, agents are visible, built-in tools are registered, and the runtime has started logging.
