Operator keeps long-term memory in markdown files. Files are the source of truth. The index is derived from those files.

That is the core design choice: memory should be readable, editable, and scoped instead of hidden behind opaque vector state.

## Two Kinds Of Memory

### Rules

Rules are always injected into the prompt.

Use a rule for short, standing instructions that should shape every future interaction.

Examples:

- "be concise in Slack replies"
- "prefer `uv` over `pip`"
- "never deploy from `main` without approval"

### Notes

Notes are searched on demand.

Use a note for durable knowledge that should not bloat every prompt.

Examples:

- staging API URL
- release schedule
- a team's recurring preference

If a fact should shape every answer, it probably belongs in a rule. If it is only sometimes relevant, it probably belongs in a note.

## Scopes

Memory lives in three scopes:

- `global`
- `agent`
- `user`

What each means:

- `global`: knowledge shared across the whole install
- `agent`: knowledge for one agent only
- `user`: private knowledge for one authenticated user

Current runtime boundary:

- `user` scope is available only in private user conversations
- public channels do not get user-scoped memory
- jobs do not run with user-scoped memory

## File Layout

```text
memory/
├── global/
│   ├── rules/
│   ├── notes/
│   └── trash/
└── users/
    └── <user>/
        ├── rules/
        ├── notes/
        └── trash/

agents/<agent>/memory/
├── rules/
├── notes/
└── trash/
```

Each memory file stores YAML frontmatter like:

```yaml
---
created_at: 2026-03-12T16:45:00Z
updated_at: 2026-03-12T16:45:00Z
expires_at: 2026-03-19T16:45:00Z
---
```

## Saving Memory

Agents use deterministic keys through the built-in tools:

- `save_rule`
- `save_note`
- `search_notes`
- `list_rules`
- `list_notes`
- `read_note`
- `forget_rule`
- `forget_note`

Example:

```python
save_rule(
    key="response-style",
    content="Use short Slack replies unless the user asks for detail.",
    scope="agent",
)

save_note(
    key="staging-api-url",
    content="https://staging.example.internal",
    scope="global",
)
```

Choose short stable keys like `response-style`, `release-date`, or `staging-api-url`.

## TTL And Expiry

Notes support optional TTL.

Supported formats:

- `30m`
- `1h`
- `3d`
- `2w`

Example:

```python
save_note(
    key="travel-week",
    content="User is traveling and may respond slowly this week.",
    scope="user",
    ttl="1w",
)
```

Expired items do not disappear silently. Operator moves them into `trash/`.

There are two behaviors to know:

- expired notes stop appearing in normal reads, lists, and searches as soon as their `expires_at` passes
- a background sweep moves expired files into `trash/` about once per hour while the runtime is running

That means expiry takes effect immediately for normal use, even if the file has not been physically moved to `trash/` yet.

## Forgetting Memory

`forget_rule` and `forget_note` do not hard-delete files. They move them into the relevant `trash/` directory and remove them from the search index.

That gives you a safer audit trail than deleting knowledge outright.

## Search

Notes are searched through SQLite FTS5 with:

- Porter stemming
- BM25 ranking

Why that matters:

- "deploy" matches "deployment"
- search stays local
- indexing is rebuildable from disk

The search index lives in:

```text
~/.operator/db/memory_index.db
```

After manual edits to memory files, run:

```sh
operator memory index
```

## Optional Embeddings

You can add vector search for notes:

```yaml
defaults:
  embeddings:
    model: "openai/text-embedding-3-small"
    dimensions: 1536
```

And install:

```sh
pip install sqlite-vec
```

When embeddings are configured, Operator stores vectors locally and fuses vector results with FTS results using Reciprocal Rank Fusion.

If `sqlite-vec` is unavailable, Operator logs a warning and continues with FTS only.

## CLI Examples

```sh
operator memory list global
operator memory list agent:operator
operator memory search "release schedule" --scope agent:operator
operator memory index
```

Use the CLI when you want to inspect or repair memory directly without going through an agent.
