Runtime config lives at `~/.operator/operator.yaml`.

## Defaults

```yaml
defaults:
  models:
    - "anthropic/claude-opus-4-6"
    - "openai/gpt-4.1"
  max_iterations: 25
  context_ratio: 0.5
  max_output_tokens: null
  timezone: "America/Vancouver"
  env_file: ".env"
```

| Field | Description |
|-------|-------------|
| `models` | Fallback chain â€” if the first model errors, the next is tried. Always use list format. |
| `max_iterations` | Maximum tool-call loops per agent run. |
| `context_ratio` | Fraction of context window to use for conversation history. |
| `max_output_tokens` | Cap output length per LLM call. `null` uses each model's max. |
| `timezone` | IANA timezone for system prompt timestamps and job scheduling. Defaults to `UTC`. |
| `env_file` | Path to env file for API keys. Relative paths resolve from `~/.operator/`. Supports `~` expansion. |

## Agents

```yaml
agents:
  operator:
    models:
      - "anthropic/claude-sonnet-4-6"
    transport:
      type: slack
      bot_token_env: SLACK_BOT_TOKEN
      app_token_env: SLACK_APP_TOKEN
```

Each agent can override `models`, `max_iterations`, `context_ratio`, and `max_output_tokens` from defaults. Agents without a `transport` block are available for jobs and sub-agent spawning but have no chat interface.

### Sandbox

```yaml
agents:
  operator:
    sandbox: false
```

| Field | Default | Description |
|-------|---------|-------------|
| `sandbox` | `true` | When `true`, file tools (`read_file`, `write_file`, `list_files`, `send_file`) are confined to the agent's workspace. When `false`, file tools accept absolute paths and paths outside the workspace. |

`run_shell` is not affected â€” use `permissions.tools` to restrict shell access.

Sub-agents inherit their parent's sandbox setting unless spawned as a different agent (via `spawn_agent(agent="...")`), in which case the target agent's sandbox setting is used.

See [Agents â€” Sandbox](/concepts/agents#sandbox) for usage guidance.

### Permissions

Restrict which tools or skills an agent can use:

```yaml
agents:
  restricted-agent:
    permissions:
      tools: [read_file, list_files, web_fetch, read_skill, run_skill, search_memories]
      skills: [summarize, translate]
```

- No `permissions` block = full access (default).
- `"*"` = explicit full access.
- `[list]` = only these names are available; everything else is hidden.
- Permissions use flat allow-lists.

<Tip>Run `operator tools` to see all available built-in tool names.</Tip>

See [Agents](/concepts/agents) for details on agent prompts and workspaces.

### Shared Directory

All agents share a `~/.operator/shared/` directory, symlinked into each workspace as `workspace/shared/`. Use this for files that need to be accessible across agents â€” shared data, output handoffs, etc. The directory is created by `operator init` and the symlink is set up automatically on each agent run.

## Roles

```yaml
roles:
  team:
    agents: [operator, researcher]
  viewer:
    agents: [researcher]
```

Roles control which agents a user can message. Manage users via the CLI (`operator user`) or the `manage_users` tool. See the [Users](/concepts/users) guide.

- `admin` is a built-in role that grants access to **all** agents. It cannot be redefined in config.
- A user's allowed agents are the **union** of all their role agent lists.

See [Users](/concepts/users) for full details on user management and role assignment.

## Memory

```yaml
memory:
  embed_model: "openai/text-embedding-3-small"
  embed_dimensions: 1536
  max_memories: 10000
  inject_top_k: 5
  inject_min_relevance: 0.1
  harvester:
    enabled: true
    schedule: "*/30 * * * *"
    model: "openai/gpt-4.1-mini"
  cleaner:
    enabled: true
    schedule: "0 3 * * *"
    model: "anthropic/claude-haiku-4-5"
```

| Field | Description |
|-------|-------------|
| `embed_model` | Required when any memory service is enabled. |
| `embed_dimensions` | Embedding vector size. |
| `max_memories` | Per-scope soft cap. |
| `inject_top_k` | Memories injected per message. |
| `inject_min_relevance` | Cosine similarity threshold. |

### Harvester

Extracts facts from conversations using an LLM and stores them as vector embeddings in SQLite (via sqlite-vec). When enabled, `schedule` and `model` are required.

### Cleaner

Deduplicates, merges, and tidies stored memories by sending them through an LLM. When enabled, `schedule` and `model` are required.

See [Memory](/concepts/memory) for more on scopes, pinning, and retrieval.

## Environment Variables

API keys are resolved from the environment, loaded via `env_file`. In the starter config generated by `operator init`, this is set to `.env` (relative to `~/.operator/`):

```sh
# LLM providers
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...

# Slack (per agent)
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
```

All model names use [LiteLLM format](https://docs.litellm.ai/docs/providers) (`provider/model-name`).

### Runtime Variables

Operator sets these environment variables automatically. They are available in `run_shell` commands, skill scripts, and job hooks:

| Variable | Description |
|----------|-------------|
| `OPERATOR_HOME` | Resolved absolute path to the Operator base directory (typically `~/.operator`). Use this instead of `~/.operator` in scripts and skill instructions â€” tilde expansion is unreliable outside shell contexts. |
| `OPERATOR_AGENT` | Name of the agent running the current job (set in job hooks). |
| `OPERATOR_DB` | Path to the SQLite database (set in job hooks). |
| `JOB_NAME` | Name of the current job (set in job hooks). |

## Settings

```yaml
settings:
  show_usage: false
  reject_response: ignore   # "announce" or "ignore"
```

| Field | Description |
|-------|-------------|
| `show_usage` | Append token usage stats as a reply after each agent response. Defaults to `false`. |
| `reject_response` | What happens when an unauthorized user messages an agent. `announce` replies with "You don't have access to this agent." `ignore` silently drops the message. Defaults to `ignore`. |
