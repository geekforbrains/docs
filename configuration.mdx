Runtime config lives at `~/.operator/operator.yaml`.

## Scan First

- `defaults.models` is required and must contain at least one LiteLLM model string.
- `runtime.timezone` must be a valid IANA timezone like `America/Vancouver`.
- `admin` is a built-in role and cannot be defined under `roles:`.
- File sandboxing defaults to `true` per agent.
- Memory is only active when `memory.harvester.enabled` or `memory.cleaner.enabled` is `true`.
- Slack is the only supported `transport.type` right now.

## Full Example

```yaml
runtime:
  timezone: "America/Vancouver"
  env_file: ".env"
  show_usage: false
  reject_response: ignore

defaults:
  models:
    - "anthropic/claude-sonnet-4-6"
    - "openai/gpt-4.1"
  thinking: medium
  max_iterations: 25
  context_ratio: 0.5
  max_output_tokens: null

agents:
  operator:
    sandbox: false
    transport:
      type: slack
      bot_token_env: SLACK_BOT_TOKEN
      app_token_env: SLACK_APP_TOKEN

  researcher:
    models:
      - "openai/o3"
    thinking: high
    permissions:
      tools: [web_fetch, read_skill, run_skill, search_memories, spawn_agent]
      skills: [skill-creator]

roles:
  team:
    agents: [operator, researcher]
  guest:
    agents: [operator]

memory:
  embed_model: "openai/text-embedding-3-small"
  embed_dimensions: 1536
  max_memories: 10000
  inject_top_k: 5
  inject_min_relevance: 0.1
  candidate_ttl_days: 14
  harvester:
    enabled: true
    schedule: "*/30 * * * *"
    models:
      - "openai/gpt-4.1-mini"
  cleaner:
    enabled: false
    schedule: ""
    models: []
```

## `runtime`

```yaml
runtime:
  timezone: "America/Vancouver"
  env_file: ".env"
  show_usage: false
  reject_response: ignore
```

| Field | Default | What it does |
| --- | --- | --- |
| `timezone` | `UTC` | Used for cron matching, Slack timestamp formatting, stable prompt timezone guidance, and request-time current-time injection |
| `env_file` | `null` | Optional `.env` file path. Relative paths resolve from the config file directory |
| `show_usage` | `false` | Sends a usage summary after each completed chat response |
| `reject_response` | `ignore` | Unknown or unauthorized user behavior: `ignore` or `announce` |

### `runtime.timezone`

Operator uses the configured timezone in two separate ways:

1. A **stable runtime block** tells the model how to interpret ambiguous dates and times.
2. A **request-only current-time block** is appended right before each live model call.

That current-time block is not persisted to conversation history.

### `runtime.env_file`

- Paths support `~` expansion.
- Relative paths resolve from the config file directory.
- Existing environment variables are not overwritten by the file.

## `defaults`

```yaml
defaults:
  models:
    - "anthropic/claude-sonnet-4-6"
  thinking: off
  max_iterations: 25
  context_ratio: 0.5
  max_output_tokens: null
```

| Field | Default | What it does |
| --- | --- | --- |
| `models` | required | Fallback chain of LiteLLM model IDs |
| `thinking` | `off` | Reasoning level: `off`, `low`, `medium`, `high` |
| `max_iterations` | `25` | Max model/tool loop iterations |
| `context_ratio` | `0.5` | Fraction of the model input window Operator is willing to spend on message history |
| `max_output_tokens` | `null` | Per-call output cap. `null` means â€śuse the model max when LiteLLM can resolve itâ€ť |

<Note>
  The starter file written by `operator init` sets `defaults.max_iterations: 50`. The model-level default, if you omit the field entirely, is `25`.
</Note>

### `models`

- Uses LiteLLM format: `provider/model-name`
- The first model is preferred
- Later models are fallbacks for live request failures

Operator also accepts a singular `model:` key in the `defaults` or per-agent config and normalizes it into `models: [...]`. The docs use list form because that is the real fallback surface.

### `context_ratio`

When history exceeds the model budget, Operator:

1. keeps the system prompt
2. keeps the latest user exchange
3. drops older exchanges first
4. shortens older non-user content if needed

Set `0` to effectively disable history trimming logic for models where LiteLLM can report token limits.

## `agents`

```yaml
agents:
  operator:
    models:
      - "anthropic/claude-sonnet-4-6"
    thinking: medium
    max_iterations: 40
    context_ratio: 0.4
    max_output_tokens: 4096
    sandbox: true
    transport:
      type: slack
      bot_token_env: SLACK_BOT_TOKEN
      app_token_env: SLACK_APP_TOKEN
    permissions:
      tools: [read_file, list_files, web_fetch]
      skills: [job-creator]
```

Each agent can override any of the `defaults` execution knobs plus transport, sandbox, and permissions.

| Field | Default | Notes |
| --- | --- | --- |
| `models` | inherit from `defaults.models` | Fallback chain for this agent |
| `thinking` | inherit from `defaults.thinking` | Simple reasoning level |
| `max_iterations` | inherit from `defaults.max_iterations` | Max tool loop depth |
| `context_ratio` | inherit from `defaults.context_ratio` | History budget fraction |
| `max_output_tokens` | inherit from `defaults.max_output_tokens` | Output token cap |
| `sandbox` | `true` | Restricts file tools to the workspace when enabled |
| `transport` | none | Makes the agent reachable from Slack |
| `permissions` | none | Optional flat allow-lists for tools and skills |

Agents without a `transport` block are headless: they can still be used for jobs and `spawn_agent(...)`, but they are not directly reachable in chat.

<Warning>
  The `operator` runtime itself still requires at least one agent with a transport. A config containing only headless agents will not start the long-running service.
</Warning>

### `sandbox`

- `true`: `read_file`, `write_file`, `list_files`, and `send_file` cannot escape the workspace
- `false`: file tools may use absolute paths or paths outside the workspace

`run_shell` is not sandboxed. Use `permissions.tools` if you need to remove shell access.

### `permissions`

```yaml
permissions:
  tools: [read_file, web_fetch, spawn_agent]
  skills: [job-creator]
```

| Value | Meaning |
| --- | --- |
| omitted | unrestricted |
| `"*"` | unrestricted |
| list | only the listed names are available |

Permissions are allow-lists. Unlisted tools and skills are removed from the LLM-visible tool/skill surface.

## `transport`

```yaml
transport:
  type: slack
  bot_token_env: SLACK_BOT_TOKEN
  app_token_env: SLACK_APP_TOKEN
```

`transport.type` must currently be `slack`.

| Field | Required | Notes |
| --- | --- | --- |
| `type` | yes | Only `slack` is supported |
| `bot_token_env` | yes | Env var name for the bot token |
| `app_token_env` | yes | Env var name for the Socket Mode app token |

Tokens are resolved from the process environment after `runtime.env_file` is loaded.

## `roles`

```yaml
roles:
  team:
    agents: [operator, researcher]
  guest:
    agents: [operator]
```

Roles define which agents a user may message.

| Rule | Behavior |
| --- | --- |
| `admin` | built-in, grants access to all agents, cannot be redefined |
| multiple roles | allowed agents are the union of all listed agent names |
| undefined custom role | grants no agent access |

`operator user add` warns when you create a non-admin user with a role name that is not defined in config.

## `memory`

```yaml
memory:
  embed_model: "openai/text-embedding-3-small"
  embed_dimensions: 1536
  max_memories: 10000
  inject_top_k: 5
  inject_min_relevance: 0.1
  candidate_ttl_days: 14
  harvester:
    enabled: true
    schedule: "*/30 * * * *"
    models:
      - "openai/gpt-4.1-mini"
  cleaner:
    enabled: true
    schedule: "0 3 * * *"
    models:
      - "openai/gpt-4.1-mini"
```

| Field | Default | Notes |
| --- | --- | --- |
| `embed_model` | `""` | Required when memory is active |
| `embed_dimensions` | `1536` | Must match the embedding model output width |
| `max_memories` | `10000` | Per-scope cap. Saves past this cap are skipped |
| `inject_top_k` | `5` | Max recalled memories after filtering |
| `inject_min_relevance` | `0.1` | Minimum cosine similarity for recall |
| `candidate_ttl_days` | `14` | TTL for candidate memories |
| `harvester` | disabled | Extracts memories from conversations |
| `cleaner` | disabled | Deduplicates and tidies stored memories |

<Warning>
  Memory is only active when `memory.harvester.enabled` or `memory.cleaner.enabled` is `true`. That same switch controls:

  - automatic memory recall in chat and jobs
  - pinned-memory injection
  - runtime memory tools like `save_memory` and `search_memories`

  The `operator memories` CLI still works because it reads the SQLite store directly.
</Warning>

### `harvester` and `cleaner`

When either worker is enabled:

- `embed_model` becomes required
- `schedule` must be a valid cron expression
- `models` must contain at least one valid LiteLLM model string

`models` is a fallback chain, just like `defaults.models`. If the first model errors or is rate-limited, the next one is tried. A singular `model:` key is also accepted and normalized into a one-element list.

The harvester reviews recent conversations and extracts reusable facts. The cleaner works scope-by-scope and can merge, split, or delete memories, but it may not delete pinned memories.

## Environment Variables Operator Sets

These variables are provided automatically in runtime contexts:

| Variable | Where | Meaning |
| --- | --- | --- |
| `OPERATOR_HOME` | shell commands, skill commands, job hooks | Absolute path to the Operator home directory |
| `JOB_NAME` | job hooks | Current job name |
| `OPERATOR_AGENT` | job hooks | Agent executing the job |
| `OPERATOR_DB` | job hooks | Absolute path to the SQLite database |

Use `$OPERATOR_HOME` instead of `~/.operator` in scripts and skills.
